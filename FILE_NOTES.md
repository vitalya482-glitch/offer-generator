# Пометки по файлам проекта

## `app.py`

Главный файл программы.

Что делает сейчас:

- читает Excel через `openpyxl`;
- пытается найти модель, количество, итоговую цену и опции;
- открывает Word-шаблон через `python-docx`;
- заменяет часть текста в КП;
- добавляет таблицу оборудования;
- добавляет таблицу опций;
- ищет PDF по названию модели в папке `pdf/`;
- сохраняет готовый Word-файл в `output/`.

Что надо улучшить:

- точно привязать чтение Excel к реальным ячейкам;
- аккуратно редактировать существующие таблицы Word, а не просто добавлять новые;
- добавить красивое форматирование в стиле исходного КП;
- сделать полноценный GUI.

Это исходный шаблон пользователя, переименованный для проекта.

Что надо сделать дальше:

- заменить `ТОО «[Организация]»` на переменную клиента;
- определить, какую таблицу нужно заполнять автоматически;

- определить место вставки технических характеристик.

Рекомендуемые переменные для будущего шаблона:

```text
{{total_sum}}

{{delivery_terms}}

{{commissioning}}

{{specification_table}}
{{technical_characteristics}}
```

## `samples/Calc_23-12-24 PAC.xlsx`

Excel-калькулятор пользователя.

Это основной источник цены и комплектации.

Что нужно уточнить/доработать:

- на каком листе лежит финальный расчет;
- где находится модель;
- где находится количество;
- где находится итоговая цена;
- где находятся опции;
- есть ли несколько валют;
- надо ли брать EXW или DDP.

## `pdf/`

Папка для PDF-файлов с техническим описанием.

Правило поиска сейчас простое:

```text
если модель CCU121A, программа ищет pdf/CCU121A.pdf
```

Можно доработать поиск так, чтобы программа искала PDF по части имени файла.

## `output/`

Сюда программа сохраняет готовые КП.

Пример результата:

```text
output/КП_ТОО_Ромашка.docx
```

## `sam_offer_generator.spec`

Теперь собирает приложение в режиме PyInstaller `onedir`:

```text
dist/SAM-Offer-Generator/
  SAM-Offer-Generator.exe
  _internal/
  config/
  modules/source/
```

Ключевое изменение: `a.binaries`, `a.zipfiles` и `a.datas` передаются в `COLLECT(...)`, а не в `EXE(...)`. Поэтому релиз получается папкой, а не одним большим EXE.

## `MODULES_MANIFEST.json`

Описывает, какие папки входят в отдельные модульные ZIP-архивы для GitHub Actions/Releases.

## `tools/package_modules.py`

Создает отдельные архивы модулей и файл `MODULES_INDEX.md`.

## `tools/prepare_portable_release.py`

После PyInstaller-сборки добавляет в portable-папку README, editable `config/`, source-копии модулей и служебный `release_info.json`.

## `requirements.txt`

Зависимости Python:

```text
python-docx
openpyxl
PyMuPDF
pyinstaller
```

## `.github/workflows/build-windows.yml`

Файл для GitHub Actions.

Он нужен, чтобы GitHub сам собирал Windows `.exe`.

После загрузки проекта в GitHub нужно открыть вкладку `Actions`.

## `config.example.json`

Пример настроек проекта.

Можно скопировать в:

```text
config.json
```

И менять значения под себя.
